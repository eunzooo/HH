    1. 시스템 개요

    본 문서는 HEAVEN OR HELL의 플레이 핵심을 이루는 세 시스템을 정의한다. 세 시스템은 독립적으로 동작하지 않으며, 하나의 판별 세션 안에서 순차적·조건부로 연결된다.

    1-1. 목적

    • 플레이어가 망자의 천국/지옥 행선지를 판정하는 판별 세션의 전체 흐름과 규칙을 구현 가능한 수준으로 명세한다.
    • 대화 분기, 정보 해금, 업경 트리거를 포함한 상호작용 규칙을 정의한다.
    • 프로그래머와 아티스트가 동일 문서를 참조하여 구현·연출 작업을 진행할 수 있도록 한다.

    1-2. 범위

    포함

    • 망자 분류 프로세스
    • 대화 시스템 규칙
    • 업경 컷신 시스템
    • 시스템 간 연동 규칙

    제외 (별도 문서)

    • 휴게실 이벤트 / 외부인 개입
    • 엔딩 분기 / 정산 시스템
    • UI/UX 레이아웃 상세
    • NPC 개별 시나리오 데이터

    연관 시스템

    • 코어 루프 (일별 근무 흐름)
    • 휴게실 이벤트 시스템
    • 엔딩 정산 시스템
    • 세이브/로드 시스템

    1-3. 용어 정의

    용어정의비고
    판별 세션하나의 망자를 프로필 확인 → 대화 → 판정까지 처리하는 단일 작업 단위1일 근무에 복수 세션 포함
    NPC 프로필이름, 나이, 직업, 사인, 범죄 이력 등 서류상 기본 정보판정의 명시적 근거
    NPC 스토리프로필만으로 드러나지 않는 맥락, 정황, 서사대화를 통해 점진적 해금

    2. 망자 분류 시스템

    판별 세션의 전체 흐름을 정의한다. 플레이어는 프로필 검토 → 대화 → (조건부) 업경 → 판정의 단계를 거친다.

    필드타입설명표시 조건

    2-3. 판정 규칙

    판정에 객관적 정답은 존재하지 않는다. 단, 내부 판정 기준표가 존재하여 성과 평가에 활용된다.

    내부 판정 기준표

    • 각 NPC에는 의도된 정답(intended_verdict)이 사전 설정된다.
    • 플레이어 판정과 의도된 정답의 일치 여부가 정확도(accuracy)로 기록된다.
    • 정확도는 엔딩 분기의 핵심 변수 중 하나이다.

    회색 지대 NPC

    • 일부 NPC는 의도된 정답이 없거나 천국/지옥 모두 유효하다.
    • 이 경우 정확도가 아닌 선택 성향(tendency)으로 분류된다.
    • 성향 누적값은 엔딩 톤에 영향을 준다. [TBD]

    2-5. 판정 결과 처리

    판정이 확정되면 아래 데이터가 기록되고, 이후 시스템에 전달된다.

    기록 항목설명소비처

    3. 대화 시스템

    망자와의 대화는 판별 세션의 핵심 상호작용이다. 플레이어의 질문 선택에 따라 정보 해금 경로가 달라지고, 업경 트리거 충족 여부가 결정된다.

    3-1. 대화 구조

    stateDiagram-v2 [*] --> 인사_단계 : 망자 입장 인사_단계 --> 자유질문 : 자동 인사 종료 자유질문 --> 자유질문 : 질문 선택 자유질문 --> 심화질문 : 핵심 키워드 해금 심화질문 --> 심화질문 : 추가 추궁 심화질문 --> 업경대기 : 트리거 플래그 충족 심화질문 --> 판정대기 : 대화 종료 선택 자유질문 --> 판정대기 : 대화 종료 선택 업경대기 --> 판정대기 : 업경 재생 완료 판정대기 --> [*] : 판정 실행

    하나의 대화 세션은 3개 단계를 순차적으로 거친다. 플레이어는 언제든 대화를 종료하고 판정으로 넘어갈 수 있다.

    단계설명진입 조건종료 조건
    인사 단계 망자가 자기소개 또는 상황 설명을 한다. 플레이어 선택지 없음. 세션 시작 시 자동 NPC 대사 종료 시 자동
    자유질문 플레이어가 선택지 풀에서 질문을 골라 대화를 이끈다. 질문 종류에 따라 NPC 반응과 해금 정보가 달라진다. 인사 단계 종료 핵심 키워드 해금 → 심화 전환
    또는 플레이어가 대화 종료 선택
    심화질문 핵심 키워드가 해금된 후 열리는 추궁 단계. 더 깊은 정보와 업경 트리거 플래그를 획득할 수 있다. 핵심 키워드 해금 트리거 충족 → 업경 재생
    또는 플레이어가 대화 종료 선택

    3-2. 질문 유형

    플레이어에게 제시되는 선택지는 아래 유형으로 분류된다. 유형에 따라 NPC 반응과 해금 결과가 달라진다.

    유형아이콘효과예시
    추궁하기 🔍 NPC의 진술 모순을 지적한다. 성공 시 숨겨진 정보 해금. 실패 시 NPC 방어 반응. "프로필에는 범죄 이력이 없다고 되어 있는데, 방금 하신 말씀과 모순되지 않나요?"
    동조하기 💬 NPC의 감정에 공감한다. 신뢰도가 올라 자발적 고백 유도. 모순 지적 기회를 잃을 수 있음. "그런 상황이었다면 누구라도 힘들었을 거예요."
    사실확인 📋 프로필 데이터와 NPC 진술을 대조한다. 중립적 반응. 정보 해금 가능성 보통. "기록에는 직업이 교사로 되어 있는데, 맞으신가요?"
    침묵 ⏸️ 아무 말 없이 기다린다. 일부 NPC는 침묵에 반응해 자발적으로 정보를 제공한다. (…)

    3-3. 대화 분기 규칙

    선택지 제시와 분기는 아래 규칙을 따른다.

    • 선택지 수: 한 턴에 2~4개 선택지를 제시한다. [TBD]
    • 선택지 구성: 최소 2개 이상의 질문 유형이 포함되어야 한다.
    • 선택지 소멸: 한 번 선택한 질문은 동일 세션에서 재등장하지 않는다.
    • 턴 제한: 자유질문 + 심화질문 합산 최대 턴 수를 두어 무한 대화를 방지한다. [TBD]
    • 대화 종료: 플레이어는 언제든 "판정으로 넘어가기" 선택지를 사용할 수 있다. 단, 인사 단계에서는 불가.

    3-4. NPC 반응 시스템

    NPC는 플레이어의 질문 유형과 대화 맥락에 따라 반응한다. 반응은 아래 카테고리로 분류된다.

    반응 유형트리거 조건결과
    솔직한 답변 신뢰도 높음 + 적절한 질문 유형 정보 플래그 해금, 스토리 진행
    회피 민감한 주제 + 신뢰도 낮음 정보 해금 실패, 다른 접근 필요
    거짓말 범죄/비밀 관련 질문 + NPC 성격 특성 잘못된 정보 제공 (추궁으로 간파 가능)
    감정 폭발 연속 추궁 또는 민감 키워드 직격 대화 흐름 단절, 심화질문 강제 전환 가능
    자발적 고백 동조 누적 + 특정 NPC 성격 조합 핵심 정보 자동 해금, 업경 트리거 기여

    3-5. 정보 해금 메커니즘

    정보 해금은 대화 시스템의 핵심 보상이자 업경 트리거의 전제 조건이다.

    flowchart LR Q1[질문 선택] --> R1{NPC 반응} R1 -- 솔직/고백 --> FLAG[정보 플래그 해금] R1 -- 회피/거짓말 --> RETRY[다른 질문 시도] RETRY --> Q1 FLAG --> CHECK{트리거 조건\n검사} CHECK -- 모든 필수 플래그 충족 --> TRIGGER[업경 트리거 활성] CHECK -- 미충족 --> Q1 CHECK -- 일부 충족 --> KEYWORD[핵심 키워드 해금\n→ 심화질문 개방] style FLAG fill:#1a3d2a,stroke:#9ed8a3,color:#e9edf4 style TRIGGER fill:#3d2e1a,stroke:#ffb38a,color:#e9edf4 style KEYWORD fill:#2a3a5c,stroke:#72c6ff,color:#e9edf4

    정보 플래그 데이터 구조

    필드타입설명
    flag_idstring고유 식별자 (예: npc_01_secret_crime)
    npc_idstring소속 NPC 식별자
    categoryenumSURFACE(표면) / DEEP(심층) / CORE(핵심)
    unlock_conditionobject해금에 필요한 질문 유형, 선행 플래그 등 조건
    display_textstring해금 시 플레이어에게 표시되는 요약 텍스트
    is_mirror_requiredbool업경 트리거의 필수 플래그 여부
    정보 플래그는 3단계 계층(SURFACE → DEEP → CORE)으로 나뉘며, CORE 플래그가 모두 해금되어야 업경이 발동한다. 플레이어는 SURFACE만으로도 판정이 가능하지만, 깊이 파고들수록 더 정확한(또는 더 흔들리는) 판단 근거를 얻게 된다.

    4. 업경 컷신 시스템

    업경(業鏡)은 NPC의 생전 행위를 영상으로 재현하는 장치이다. 대화의 절정부에서 발동하며, 판별의 감정 밀도를 극대화하는 서사 보상 메커니즘이다.

    4-1. 트리거 조건

    업경은 자동 발동이며, 플레이어가 임의로 실행할 수 없다.

    조건 유형설명필수 여부
    CORE 플래그 충족 해당 NPC의 is_mirror_required = true인 모든 정보 플래그가 해금된 상태 필수
    대화 단계 심화질문 단계에 진입한 상태 필수
    특수 NPC 조건 일부 NPC는 추가 조건이 존재 (예: 특정 질문 유형 사용, 특정 선택지 경유) NPC별 선택 [TBD]
    flowchart TD subgraph 트리거_검사["트리거 조건 검사"] C1[CORE 플래그\n전부 해금?] --> C2[심화질문\n단계인가?] C2 --> C3[특수 조건\n충족?] end C3 -- 모두 충족 --> FIRE[업경 발동] C3 -- 하나라도 미충족 --> SKIP[업경 미발동\n→ 대화 계속 or 판정] FIRE --> TRANS[화면 전환 연출] TRANS --> PLAY[컷신 재생] PLAY --> RETURN[대화 화면 복귀] RETURN --> DECIDE[판정 화면 전환] style FIRE fill:#3d2e1a,stroke:#ffb38a,color:#e9edf4 style SKIP fill:#2a2a2a,stroke:#666,color:#aab4c3

    4-2. 연출 흐름

    업경 발동 시 아래 순서로 연출이 진행된다.

    순서단계연출 내용예상 시간
    1 진입 연출 화면이 백색/흑색으로 페이드. 업경 프레임 UI가 등장. NPC 대사 음성이 페이드아웃. 1~2초 [TBD]
    2 컷신 재생 NPC의 생전 장면을 일러스트 + 텍스트 + 효과음으로 재현. 복수 장면(scene)이 순차 재생된다. 15~45초 [TBD]
    3 핵심 공개 컷신의 마지막 장면에서 NPC의 진실 또는 반전이 드러난다. 전용 연출(확대, 색조 변화 등)이 적용된다. 3~5초
    4 복귀 연출 업경 프레임 소멸. 대화 화면으로 페이드 복귀. NPC가 짧은 반응 대사를 출력한다. 1~2초
    5 판정 전환 업경 재생 직후 판정 화면으로 자동 전환. 추가 대화 불가. 즉시

    4-3. 컷신 장면 데이터 구조

    하나의 업경 컷신은 복수의 장면(scene) 시퀀스로 구성된다.

    필드타입설명
    mirror_idstring컷신 고유 식별자 (예: mirror_npc_01)
    npc_idstring대상 NPC 식별자
    required_flags[]string[]발동에 필요한 CORE 플래그 ID 목록
    scenes[]Scene[]순차 재생되는 장면 배열
    post_dialoguestring업경 종료 후 NPC 반응 대사

    Scene 객체 구조

    필드타입설명
    scene_indexint재생 순서 (0부터 시작)
    illustrationstring배경 일러스트 리소스 경로
    narrationstring나레이션 텍스트
    sfxstring효과음 리소스 경로 (nullable)
    durationfloat표시 시간(초). 0이면 플레이어 입력 대기.
    transitionenumFADE / CUT / DISSOLVE [TBD]
    is_revealbooltrue면 핵심 공개 장면 — 전용 연출 적용

    4-4. 스킵 및 예외 처리

    • 첫 회차: 업경 컷신은 스킵 불가. 플레이어에게 핵심 서사 보상을 강제로 전달한다.
    • 2회차 이후: 이미 본 컷신은 스킵 가능 여부 검토. [TBD]
    • 미발동 세션: 업경이 발동하지 않아도 판정은 정상 진행된다. 미발동 시 판정 화면에 "업경 미확인" 표시를 둘지 여부 검토. [검토필요]
    • 세이브/로드: 업경 재생 중 세이브는 불가. 업경 시작 직전 오토세이브 검토. [TBD]

    6. 예외 처리

    정상 흐름을 벗어나는 상황에 대한 처리 규칙을 정의한다.

    상황처리 규칙비고
    인사 단계에서 판정 시도 불가. "판정으로 넘어가기" 선택지가 인사 단계에서는 비활성화. UI 차원 비활성
    대화 없이 즉시 판정 자유질문 1턴 이상 진행 후에만 판정 가능. [TBD] 최소 상호작용 보장
    업경 재생 중 강제 종료 업경은 중단 불가. 게임 자체 종료 시 업경 시작 직전 시점으로 복구. 오토세이브 연동
    모든 선택지 소진 더 이상 질문이 없으면 자동으로 판정 화면 전환. 턴 제한과 별도
    회색 지대 NPC + 업경 미발동 정상 처리. 의도된 정답이 없으므로 accuracy_match = null 기록. -
    외부인 메모가 있는 NPC 프로필 비고란에 메모 표시. 대화 선택지에 메모 관련 질문 추가 여부 [TBD] 휴게실 연동

    7. 구현 메모

    프로그래밍 구현 시 참고할 기술적 메모.

    대화 데이터 관리

    • NPC별 대화 트리는 외부 데이터 파일(CSV 또는 JSON)로 관리하여 기획자가 직접 수정 가능하게 구성한다.
    • 정보 플래그와 트리거 조건도 동일 데이터 파일 내에 정의한다.
    • Unity 기준: ScriptableObject 또는 Addressable로 NPC 데이터 로드. [WIP]

    업경 연출

    • 컷신은 런타임 생성이 아닌 사전 제작 시퀀스로 처리한다.
    • Unity Timeline 또는 커스텀 시퀀서를 사용하여 일러스트 + 텍스트 + SFX를 동기화한다. [WIP]
    • 리소스는 Addressable 비동기 로드 — 업경 트리거 감지 시점에 프리로드 시작.

    상태 관리

    • 판별 세션 상태는 SessionState 클래스로 중앙 관리한다.
    • 상태 전이: PROFILE → DIALOGUE → (MIRROR) → VERDICT → RESULT
    • 각 전이 시점에 이벤트를 발행하여 UI, 오디오, 연출이 구독.

    세이브 연동

    • 세이브 시점: 판별 세션 시작 전 + 판정 완료 후
    • 업경 재생 중 세이브 불가 — 업경 시작 직전 오토세이브로 보장.
    • 세이브 데이터에 포함: 일차, 세션 인덱스, 누적 판정 기록, 해금 플래그 전체 [WIP]
