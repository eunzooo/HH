    00 문서 개요

    0.1 작성 목적 및 범위

    본 문서는 Heaven or Hell의 개발을 위한 시스템 기획서로, 게임을 구성하는 핵심 시스템의 동작 방식, 데이터 구조, 분기 로직을 명세한다.

    • 코어 루프 및 일차(Day) 구조 정의
    • 망자 분류·대화·업경 시스템 상세 명세
    • 직원 휴게실 및 외부인 개입 시스템 명세
    • NPC 데이터 구조 및 생성 규칙 정의
    • 업보 수치·엔딩·해금 시스템 정의
    • UI/UX 화면 전환 흐름 정의
    • DLC 확장 설계 방향 제시

    0.2 용어 정의

    용어정의
    망자(亡者)분류 대상이 되는 NPC. 플레이어가 천국 또는 지옥을 판정해야 한다.
    업경(業鏡)망자와의 대화 중 특정 플래그 조건 달성 시 발동하는 스토리 컷신. 해당 망자의 생전 주요 장면을 보여준다.
    외부인직원 휴게실에 등장하는 특수 NPC(저승사자·신·크툴루·파라오 등). 플레이어에게 청탁을 건넨다.
    Day1회 출근~퇴근의 단위. 총 10회 반복 후 엔딩 처리된다.
    업보 수치(業報數値)엔딩 항목별로 독립적으로 증감하는 수치. 망자 분류 및 휴게실 상호작용 모두 이 수치를 변동시킨다. 게임 종료 시 가장 높은 항목의 엔딩이 발동된다. (구 명칭: 인사평가 점수)
    업보 항목업보 수치를 구성하는 개별 카테고리. 엔딩 수만큼 존재한다. (예: 신 수치, 악마 수치, 평범 수치 등)
    청탁외부인이 플레이어에게 요구하는 특정 분류 지시. 수락·거절 여부가 업보 수치에 영향.
    코어 루프출근 → 망자분류 씬 → 직원휴게실 씬 → 퇴근으로 이어지는 1일치 게임플레이 사이클.
    분류 씬망자를 상대하는 메인 플레이 공간. 프로필 열람, 대화, 판정 버튼이 모두 이 씬에서 동작한다.
    휴게실 씬업무 후 외부인과 상호작용하는 공간. 분류 씬으로 건드릴 수 없는 업보 수치 항목을 증감하거나 특정 엔딩 경로를 해금한다.

    2.2 기능 1 — 판정 버튼

    ✅ 확정
    플레이어는 망자를 상대하는 도중 언제든지 천국 또는 지옥 버튼을 누를 수 있다. 대화 종료 조건 없음.
    항목내용
    활성 조건분류 씬 진입 시부터 상시 활성 (업경 컷신 재생 중 비활성)
    판정 실행버튼 클릭 → 망자 분류 방향 애니메이션 재생 → 다음 망자로 전환
    업보 수치 반영망자별 정답 여부에 따라 해당 업보 항목 즉시 증감
    오판정 예시착한 망자를 지옥으로 보낼 경우 → 악마 수치↑ / 신 수치↓
    버튼 디자인천국(↑) / 지옥(↓) 도장 스탬프 연출 포함 (§10.3 UI_STAMP 참조)

    2.3 기능 2 — 프로필 열람

    ✅ 확정
    망자 스프라이트를 클릭하면 이력서/입국심사서류 형태의 프로필 팝업이 표시된다. 업경 컷신 재생 중에는 비활성.
    단계동작
    ① 망자 입장스프라이트가 화면 바깥 아래에서 책상 위(화면 중앙 하단)로 이동하며 배치
    ② 스프라이트 클릭망자 스프라이트를 화면 바깥으로 이동, 화면 중앙에 망자 프로필 스프라이트 표시
    ③ 프로필 팝업이력서·입국심사서류와 유사한 문서 디자인. 이름·나이·직업·사인·죄목·선행 등 텍스트+스프라이트 표시
    ④ 팝업 닫기프로필 스프라이트 바깥 영역 클릭 → 디폴트 상태(망자 스프라이트 책상 위) 복귀
    ⑤ 비활성 조건업경 컷신 재생 중에는 클릭 무효

    2.4 기능 3 — 대화창

    ✅ 확정
    화면 중앙에 메신저 앱 형태로 대화 출력. 스크롤로 이전 대화 열람 가능.

    03 직원 휴게실 시스템

    3.0 휴게실 씬의 역할 (확정)

    ✅ 확정 — 휴게실의 두 가지 핵심 기능
    1. 업보 수치 직접 조작 — 망자 분류만으로는 건드릴 수 없는 업보 수치 항목을 증감시킨다. 외부인 청탁이 주요 수단.
    2. 엔딩 경로 해금 — 특정 엔딩 항목은 기본적으로 비활성 상태이며, 휴게실 상호작용으로 잠금 해제되어야 해당 엔딩에 도달 가능하다.

    모든 NPC·대화·엔딩 데이터는 CSV 파일로 외부 관리한다. 코드 수정 없이 데이터 파일만 교체하면 신규 NPC·DLC 콘텐츠를 즉시 반영할 수 있다. Unity의 Resources.Load 또는 StreamingAssets 경로로 런타임 로드.

    09 확장성 설계 (DLC)

    9.1 기본 시스템 확장 구조

    📌 설계 원칙
    메인 시스템(망자 분류·대화·판정·엔딩)을 그대로 유지하면서, 캐릭터·다이얼로그·시나리오 데이터만 교체·추가함으로써 낮은 비용으로 빠른 확장이 가능하다.

    확장 가능 요소

    • 새로운 망자 NPC 풀 추가
    • 신규 외부인 캐릭터 추가
    • 신규 엔딩 추가
    • 특수 시나리오 모드

    변경 불필요 요소

    • 대화 시스템 코어
    • 업경 컷신 시스템
    • 업보 수치 집계 로직
    • UI/UX 기본 레이아웃

    9.2 DLC 예시 설계

    DLC 1 — 신칸센 대폭발 사건

    열차 탈선 대형사고로 다수의 망자가 한꺼번에 유입. 모든 NPC가 하나의 사건에 연루된 옴니버스 구조로, 기존 독립 에피소드 방식에서 추리물 장르로 확장한다.

    • 핵심 변경: 망자 간 관계 데이터 필드 추가 필요
    • 신규 시스템: 용의자 지목 / 사건 재구성 화면 TBD
    • 기존 시스템 활용: 대화·업경·판정 시스템 그대로 사용

    DLC 2 — 할로윈 소동

    할로윈 날 망자가 현실로 나갈 수 있는 특수 이벤트 설정. 기존 인간 망자 중심에서 판타지 존재(유령·마녀·흡혈귀 등)로 캐릭터 방향 전환 가능.

    • 핵심 변경: 판정 기준 확장 (현실 출입 허가/불허)
    • 캐릭터 디자인 방향: 기존 카툰풍 유지, 할로윈 테마 적용
    • 기존 시스템 활용: 대화·청탁·엔딩 시스템 그대로 사용
